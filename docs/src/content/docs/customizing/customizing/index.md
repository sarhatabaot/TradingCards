title: "Customizing"
---

:::note
It is recommended you use the commands to create/edit/delete anything you make. You can however directly edit the files in YAML mode if that's easier for you. Just don't forget to save your changes and run `/cards reload` when you are done.
:::

This section covers the objects players actually collect and interact with:

- [Cards](./cards.md) for individual collectibles
- [Rarities](./rarities.md) for color, weighting, and reward structure
- [Series](./series.md) for grouping cards
- [Custom Types](./custom-types.md) for drop source categories
- [Upgrades](./upgrades.md) for progression mechanics
- [Drop Pools](./drop-pools.md) for controlled random selection

For most servers, the recommended workflow is:

1. Create rarities and series first.
2. Configure your base materials and display settings.
3. Create cards and assign them to the correct rarity and series.
4. Reload after direct YAML edits, or use commands to avoid syntax mistakes.
