Version 5.4 brought many config changes to the plugin. **It is recommended to backup all your files before moving to this version.**

## What Changed

After upgrading, the old single `config.yml` layout is replaced with multiple files:

- `general.yml`
- `chances.yml`
- `packs.yml`
- `rarities.yml`
- `series.yml`

## Migration Checklist

1. Back up your full plugin folder before upgrading.
2. Install the newer plugin build.
3. Compare your old settings against the new split files.
4. Move pack, rarity, and series data into the new file layout.
5. Start the server and verify that cards, packs, and drops still load correctly.

If you are upgrading from a very old install, expect some manual cleanup instead of a fully automatic migration.
