Use this migration when you are moving card data from YAML files into SQL-backed storage.

## Steps

1. Back up your plugin folder and database.
2. Configure your SQL connection in [`storage.yml`](../customizing/settings/storage.md).
3. Run `/cards migrate data`.
4. Restart your server.
5. Confirm that cards, rarities, series, and packs load from SQL before removing old YAML backups.

:::note
Do not switch a live production server to SQL without verifying the connection details first. A bad database configuration will block loading and make troubleshooting harder than keeping YAML enabled until migration is complete.
:::
